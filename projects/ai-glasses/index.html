🚀 项目背景：超越“聊天”的行动辅助#

为什么要做这个项目？

目前市面上的 AI 眼镜大多停留在“GPT 对话”层面，缺乏对物理世界的感知能力。本项目旨在构建一个真正的视障人士行动辅助系统 。

它不仅能“看”到世界，还能理解复杂的环境——比如识别盲道方向、判断红绿灯状态，甚至通过语音指令帮你寻找“找不到的水杯”。#

🏗️ 系统架构：端云协同#

为了在保证轻量化的同时实现复杂的视觉算法，我采用了分体式架构：

  1. 佩戴端 (采集层)：极致轻量，负责采集视听数据。
  2. 服务端 (算力层)：由一台高性能 PC (RTX 3060) 承担，处理 YOLO 分割与大模型推理。

🛠️ 硬件选型 (Hardware)#

1. 佩戴端 (Wearable)#

  • 核心主控: ESP32-CAM (低成本 WiFi 视频传输方案)
  • 视觉传感器: OV2640 摄像头模块
  • 姿态传感: MPU6050 IMU (用于头部姿态解算，辅助 3D 空间定位)
  • 音频交互: I2S 麦克风 + 骨传导耳机 (实现全双工语音通话)
  • 供电: 3.7V 锂电池 + 升压模块

2. 计算服务端 (Server)#

由于算法涉及图像分割和多模态大模型，无法在树莓派上运行，因此搭建了基于 NVIDIA RTX 3060 (12GB) 的本地推理服务器。

🧠 核心算法与软件栈 (Tech Stack)#

深度学习模型 (Local & Cloud)#

为了实现“全能”辅助，我部署了一套复杂的模型流水线：

  • 环境感知 (Nav): 使用 YOLO-Seg 进行盲道与斑马线的实时分割，结合 TrafficLight Model 识别红绿灯。
  • 万物检测 (Find): 部署 YOLO-World (Open Vocabulary Detection)，实现了对任意物体（如“药瓶”、“钥匙”）的开放词汇检测，无需预训练特定类别。
  • 人机交互 (HCI):
    • ]ASR: 阿里云 DashScope (Paraformer 模型) 实现语音转文字。
    • LLM: 调用 Qwen-Omni-Turbo (通义千问) 处理多模态理解与生成自然语言回复。
    • 手势: Google MediaPipe Hand 用于手部追踪，辅助抓取物体。

软件架构#

  • 后端框架: Python FastAPI (处理 WebSocket 高频视频流)。
  • 通信协议: WebSocket (ESP32 -> PC 传输 MJPEG 流与 PCM 音频)。

💡 核心功能实现#

1. 盲道导航与避障#

通过 YOLO-Seg 分割出盲道区域，计算掩膜中心线。当用户偏离盲道时，系统会通过骨传导耳机发出“向左偏”、“直行”等指令 。

2. “帮我找东西” (Visual Grounding)#

用户发出语音指令：“帮我找一下桌上的红牛”。

  1. 系统解析语音意图。
  2. 调用 YOLO-World 检测视野内的“红牛”物体。
  3. 结合 MPU6050 姿态数据与手势识别，语音引导用户手部移动：“再往左一点，就在手边”。

3. 全双工语音助手#

区别于传统的“唤醒词”模式，系统支持全双工 (Full-Duplex) 对话，用户可以随时打断 AI 的播报，交流更加自然流畅。

📸 监控与调试#

开发了一个 Web 监控台，可以在浏览器中实时查看：

  • ESP32 传回的第一视角画面
  • YOLO 实时标注的检测框与分割掩膜
  • 设备的 IMU 姿态数据
  • 后台日志与“黑匣子”录像

📝 难点攻克#

1. CUDA 环境配置#

由于依赖 YOLO-World 和各类分割模型，配置 PyTorch 的 GPU 环境极其复杂。需要精确匹配 CUDA 11.8+ 与 Torch 版本，否则无法调用 RTX 3060 加速。

2. ESP32 音视频同步#

ESP32 性能有限，同时开启 WiFi 视频流 (MJPEG) 和音频流 (PCM) 极易造成卡顿。最终通过优化 WebSocket 封包策略和降低 JPEG 质量解决了高延迟问题。
